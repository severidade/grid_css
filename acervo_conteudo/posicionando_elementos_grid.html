Posicionando elementos no grid

Conhecendo os termos

Grid Track é o espaço entre duas grid lines. É definido no grid explícito usando grid-template-columns e grid-template-rows

Grid Lines são verticais e horizontais e podem ser referidas através de um número ou pelo nome. Quando definido um valor para o grid-gap elas ficam mais largas.

Grid Cell É a menor unidade em um grid delimitada por quatro grid lines comparado a uma célula de uma tabela.

Grid Area Qualquer área no grid definida por quatro grid lines que contenha mais do que uma grid-cell

Criando um Grid Base

Para exemplificar estou criando um grid com três colunas e adicinando colunas outomáticas com altura de 200px usando a propriedade grid-auto-rows.

Exemplo 09
				
.cards { 
	display: grid;
	grid-template-columns: repeat(3, 1fr);
	grid-auto-rows: 200px;
	grid-gap: 10px;
}

Posicionando usando Line Numbers

Os exemplos 10 a 12 definem para o elemento 1 sua grid-area com base na numeração das colunas. O resultado é o mesmo mas a forma de declarar varia.

No Exemplo 10 a área é declarada usando as propriedades grid-column-start, grid-column-end, grid-row-start e grid-row-end.

Exemplo 10
				
.cards { 
	display: grid;
	grid-template-columns: repeat(3, 1fr);
	grid-auto-rows: 200px;
	grid-gap: 10px;
}	
.cards .item:nth-child(1) {
	grid-column-start: 2;
	grid-column-end: 4;
	grid-row-start: 1;
	grid-row-end: 3;
}

Posicionando usando Grid-area

A decralação mais simples feita referenciando o número das linhas que delimitam a área que o grid item devera ocupar é feita através da propriedade grid-area com os valores correspondentes ao grid-row-start, grid-column_start, grid-row-end e grid-column-end separados por "/".

Ao contrário da declaração de padding os valores usados correspondem a declaração dos limites da área no sentido anti-horário.

Exemplo 12
				
.cards { 
	display: grid;
	grid-template-columns: repeat(3, 1fr);
	grid-auto-rows: 200px;
	grid-gap: 10px;
}	
.cards .item:nth-child(1) {
	grid-area: 1 / 2 / 3 / 4;
}

Posicionando usando line names

Quando o grid é criado as linhas devem ser nomeadas. Assim os grid-itens poderão ser posicionados no grid das seguintes formas:

  • Mencionando o nome das linhas que delimitam o início e o fim da área do grid-item com as propriedades grid-column-start, grid-column-end, grid-row-start e grid-row-end.
  • Usando a propriedade grid-area e mencionando as linhas de inicio e fim através dos nomes das respectivas e assim delimitando a área do grid-item
  • Usando a propriedade grid-area e mencionando a área como no caso da célula 3. No exemplo as linhas definem dois conteiners um com o nome content e outro com o nome main.
Exemplo 13
				
.cards { 
	display: grid;
	grid-template-columns: [main-start] 1fr [content-start] 1fr [content-end] 1fr [main-end];
	grid-template-rows: [main-start] 100px [content-start] 100px [content-end] 100px [main-end];
	grid-gap: 10px;
}	
.cards .item:nth-child(1) {
	grid-row-start: content-end;
	grid-column-start: content-end;
	grid-row-end: main-end;
	grid-column-end: main-end;
	
}
.cards .item:nth-child(2) {
	grid-area: main-start / main-start / main-end / content-start;	
}

.cards .item:nth-child(3) {
	grid-area: content ;
}

Definindo Grid áreas

Cada grid-iem é nomeado com o uso da propriedade grid-area. No grid-conteiner, com o uso da propriedade grid-template-areas, mapeamos o posicionamento dos itens no grid.

Usando Media queries podemos definir grids e posicionamentos diferenciados para os itens em função dos tamanhos de telas.

Exemplo 14
				
.cards{
	display: grid;
	grid-template-columns: repeat(3 , 1fr);
	grid-template-rows: repeat(2 , 200px);
	grid-gap: 10px;
	grid-template-areas:	" side1 main main"
							" side2 main main";
}
.cards .item:nth-child(1) {
	grid-area: side1;
	
}
.cards .item:nth-child(2) {
	grid-area: side2;	
}

.exemplo_14 .item:nth-child(3) {
	grid-area: main;
}
